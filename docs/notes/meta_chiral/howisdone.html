¿Cómo replicar este proyecto?

Bases del proyecto

Este proyecto ha sido realizado utilizando varias herramientas open source. Quizás la más importante es llamada Quarto que es un software open-source para la publicación de documentos científicos que está construido en Pandoc. Quarto permite la ejecución de código y, por esta característica, la generación de documentos es dinámica y diversa. Otra característica importante es que Quarto permite la exportación de documentos en múltiples formatos, como HTML, PDF (utilizando algún engine de \LaTeX), Word, etcetera.

Uno de los objetivos de este proyecto es conseguir una publicación dual en los formatos de HTML y \LaTeX. Aunque Quarto permite la exportación a ambos documentos está no esta totalmente integrada y es común que un formato de qmd pueda exportarse perfectamente a HTML pero no a \LaTeX y viceversa.

Inclusión de formulas químicas en HTML

La inclusión de ecuaciones para reacciones químicas en latex se puede conseguir importando el paquete llamado mhchem. Dicho código no funcionará al exportarse a HTML debido a las ecuaciones se realizan con el engine de render asignado por Quarto. Para conseguir está característica, se ha utilizado el engine de Katex con un la extensión de mhchem para katex. Para hacerlo funcionar únicamente se debe agregar al archivo .yml lo siguiente:

format:
  html:
    html-math-method: katex
    header-includes: <script defer src="https://cdn.jsdelivr.net/npm/katex@0.16.2/dist/contrib/mhchem.min.js" integrity="sha384-RTN08a0AXIioPBcVosEqPUfKK+rPp+h1x/izR7xMkdMyuwkcZCWdxO+RSwIFtJXN"  crossorigin="anonymous"></script>

Inclusión de macros en \KaTeX

La inclusión de macros en \KaTeX es realizada por el mismo \KaTeX, véase la documentación. La forma ideal de incluir las macros es ocultarlas en un bloque de .hide para evitar la propagación de espacios. Por ejemplo, un macro utilizado en la pagina es

:::{.hide}
$$
      \gdef\diff#1#2{\frac{d#1}{d#2}} 
      \gdef\pdiff#1#2{\frac{\partial #1}{\partial #2}} 
      \gdef\con#1{[\ce{#1}]}
$$
:::

el cual es cargado en las paginas que sea necesitado con {{< include macro.qmd >}}.
